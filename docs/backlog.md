# Backlog Inicial — Épicos e User Stories

Este documento apresenta a organização inicial do backlog do sistema **FinansMe**, estruturado por **Épicos** e **User Stories**, com base no escopo funcional inicial do projeto.

---

## Épico 1 — Autenticação e Acesso

**Descrição:**  
Este épico contempla as funcionalidades necessárias para que o usuário possa criar uma conta, autenticar-se no sistema e acessar com segurança sua área financeira.

### User Stories 1

- **US01 — Cadastro de usuário**  
  *Como visitante, quero criar uma conta no sistema para começar a registrar minha vida financeira.*

- **US02 — Login no sistema**  
  *Como usuário, quero realizar login com e-mail e senha para acessar meus dados financeiros com segurança.*

- **US03 — Controle de status da conta**  
  *Como sistema, quero controlar o status de atividade do usuário para permitir apenas acessos válidos.*

---

## Épico 2 — Gestão de Contas Financeiras

**Descrição:**  
Este épico reúne as funcionalidades de cadastro e manutenção das contas do usuário, como contas bancárias e carteiras, permitindo o controle dos valores disponíveis.

### User Stories 2

- **US10 — Cadastro de conta financeira**  
  *Como usuário, quero cadastrar uma conta financeira para organizar onde meu dinheiro está armazenado.*

- **US11 — Listagem de contas financeiras**  
  *Como usuário, quero visualizar minhas contas cadastradas para acompanhar minhas fontes de saldo.*

- **US12 — Atualização de conta financeira**  
  *Como usuário, quero editar os dados de uma conta para manter minhas informações financeiras atualizadas.*

---

## Épico 3 — Gestão de Categorias e Transatores

**Descrição:**  
Este épico cobre os cadastros complementares necessários para os lançamentos financeiros, como planos de conta e transatores.

### User Stories 3

- **US20 — Cadastro de plano de conta**  
  *Como usuário, quero cadastrar categorias de receita e despesa para organizar melhor meus lançamentos.*

- **US21 — Cadastro de transator**  
  *Como usuário, quero cadastrar transatores para identificar com quem realizei uma movimentação financeira.*

- **US22 — Consulta de cadastros complementares**  
  *Como usuário, quero visualizar meus planos de conta e transatores para utilizá-los nos lançamentos financeiros.*

---

## Épico 4 — Registro de Movimentações Financeiras

**Descrição:**  
Este épico contempla o lançamento das movimentações financeiras do sistema, permitindo registrar receitas e despesas com seus respectivos detalhes.

### User Stories 4

- **US30 — Registro de movimentação financeira**  
  *Como usuário, quero cadastrar uma movimentação financeira para registrar uma receita ou despesa.*

- **US31 — Informar dados completos do lançamento**  
  *Como usuário, quero informar conta, categoria, transator, datas e valor no lançamento para manter meu controle financeiro detalhado.*

- **US32 — Definir tipo e forma de pagamento**  
  *Como usuário, quero definir o tipo de movimento e a forma de pagamento para classificar corretamente cada lançamento.*

---

## Épico 5 — Controle de Situação Financeira

**Descrição:**  
Este épico trata do acompanhamento do status das movimentações financeiras, permitindo ao usuário saber o que está em aberto, quitado ou vencido.

### User Stories 5

- **US40 — Identificação da situação financeira**  
  *Como usuário, quero definir a situação financeira de um lançamento para saber se ele está em aberto, quitado ou vencido.*

- **US41 — Atualização do status de pagamento**  
  *Como usuário, quero atualizar a situação de uma movimentação para refletir seu pagamento ou vencimento.*

- **US42 — Consulta por situação**  
  *Como usuário, quero filtrar movimentações por situação financeira para acompanhar meus compromissos pendentes.*

---

## Épico 6 — Acompanhamento e Organização Financeira

**Descrição:**  
Este épico reúne funcionalidades voltadas para a visualização e organização da vida financeira do usuário, promovendo mais responsabilidade e clareza sobre receitas e despesas.

### User Stories 6

- **US50 — Visualização do histórico financeiro**  
  *Como usuário, quero visualizar meu histórico de movimentações para acompanhar minha evolução financeira.*

- **US51 — Resumo de receitas e despesas**  
  *Como usuário, quero visualizar um resumo das minhas receitas e despesas para entender melhor minha situação financeira.*

- **US52 — Organização da vida financeira**  
  *Como usuário, quero manter meus lançamentos organizados para tomar decisões financeiras com mais responsabilidade.*

---

## Observações

- Cada épico pode ter quantas stories forem necessárias.  
- O backlog inicial contém apenas **épicos + primeiras stories** em alto nível.  
- As histórias seguem o padrão:  
  **Como [ator], quero [intenção] para [benefício].**
